1. NUnit

3. MSTest (Microsoft's testing framework)

NUnit: NUnit is a popular open-source testing framework for .NET applications.
It provides a rich set of assertions and test runners, making it easy to write and execute
unit tests. NUnit supports parameterized tests, test fixtures, and various test attributes 
to control test execution.

Key NUnit Attributes:
[TestFixture] => Used to denote a class that contains tests.
[Test] => Used to denote a method that is a test case.
[SetUp] => Used to denote a method that should be run before each test.
[TearDown] => Used to denote a method that should be run after each test.
[TestCase] => Used to denote a method that should be run with specific parameters.
eg: [TestCase(1, 2, 3)]
[Ignore] => Used to denote a test that should be ignored during execution.

Assertions in NUnit:
Assert will be used to verify the expected outcome of a test case. 
Some common assertions include:
Assert.AreEqual(expected, actual) => Verifies that two values are equal.
Assert.AreNotEqual(expected, actual) => Verifies that two values are not equal.
Assert.IsTrue(condition) => Verifies that a condition is true.
Assert.IsFalse(condition) => Verifies that a condition is false.
Assert.IsNull(object) => Verifies that an object is null.
Assert.IsNotNull(object) => Verifies that an object is not null.
Assert.Throws<TException>(TestDelegate) => Verifies that a specific exception is thrown.


AAA => Arrange => Set up the necessary objects and conditions for the test.
AAA => Act => Execute the method or functionality being tested.
AAA => Assert => Verify the expected outcome of the test.
